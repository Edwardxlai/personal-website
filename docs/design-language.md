# Design Language · 编辑×系统 (Editorial × System)

> 本次视觉方向的实现真源。DESIGN.md 是完整原始规范；本文件记录「编辑×系统」方向落地后的关键决策与组件规则。

## VI 摘要

- **类型**：个人作品站 / portfolio（不是产品落地页）。主角是「具体的人 + 三个项目构成的一条主线」。
- **主线（叙事核心）**：`流转 → 进入 → 涌现`——内容从「让已有内容流动」到「让静态文本可进入」到「让内容从角色互动中涌现」。三个项目不是并列卡片，而是一条递进。
- **气质三词**：精确 · 克制 · 有生命感。
- **信号色**：单一酸黄绿 `--accent oklch(0.89 0.215 125)`，占比 <10%，只用于身份点、主线词 hover、关键链接、Hero 主张收束点。禁止第二强调色、禁止紫蓝渐变、禁止玻璃拟态泛滥。
- **画布**：深色单主题，靠亮度层级建立深度（见 tokens 的 layer-1/2/prominent），不用黑色大阴影。

## 字体系统（已自托管 · fontsource）

| 角色 | 字体 | 用途 |
|---|---|---|
| 拉丁展示 | **Archivo Variable** (wght) | 超大姓名、英文标题、项目名、导航 |
| 中文编辑巨标题 | **Noto Serif SC** 700/900 | Hero 主张、项目 tagline、主线陈述句 |
| 中文正文 | **Noto Sans SC** 400/500/700 | 正文、说明 |
| 序号/标签/规格 | **JetBrains Mono Variable** | 巨号序号、S/01 索引、规格表 dt、元数据 |

字号层级见 `src/styles/tokens.css`（`--fs-thesis` 巨型主张 / `--fs-numeral` 巨号序号 / `--fs-hero` 拉丁姓名 …）。

## 组件规则（系统化）

- **Hero**：身份条（mono，姓名 + AVAILABLE 点）→ 巨型中文主张（serif 900，问题即主张，末字带酸绿方块收束点）→ lead → 系统化项目主线索引（`01 流转 → 02 进入 → 03 涌现`，mono 序号 + hover 变酸绿）→ 元数据条。
- **区块标题**：mono 英文 eyebrow（酸绿）+ 右侧 `S / 0x` 索引 + 发丝分隔线 + 大号中文标题。统一节奏，避免每段都长一样。
- **项目卡（ProjectFeature）**：巨号序号 `01` + 主线词 + 项目名顶部成排 → 左右交错的「真实截图 + 规格表」。规格表用 `ROLE / STACK / RESULT` 的 mono 键值行（系统感）。tagline 用 serif。封面下挂 3 张 gallery 缩略图。
- **图片**：全部走 Astro `<Image>`（webp/avif + 固定宽高，消除布局偏移）。圆角 18px + 1px 内描边，不用统一悬浮阴影。
- **按钮**：主按钮酸绿胶囊；次按钮文字 + 箭头（hover 箭头位移），不用透明描边胶囊。
- **动效**：第一版仅 `.reveal` 进入 + hover 反馈 + `prefers-reduced-motion` 降级。标志性交互（React Bits Hero）留待 Phase 3。

## 反模式（本项目明确不做）

- 首页做成组件展览 / React Bits 合集。
- 每个区块都是「标题 + 发丝线 + 列表」的同一形状（这是上一版「单调」的根因，已用巨号序号 / serif 主张 / 规格表 / 交错图打破）。
- 三列等宽项目卡、技能进度条、九宫格填充、签名字体、第二强调色。
- 伪造用户数 / 收入 / 融资 / 奖项；A2A 等概念误用（搭映是角色 Agent 协作，非 Google A2A）。
